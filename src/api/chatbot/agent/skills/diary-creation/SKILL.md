# 日記作成スキル

## いつ使うか

- ユーザが「日記を書きたい」「今日のことを記録して」「日記にまとめて」などと言ったとき
- 会話の中でユーザの出来事を日記形式でまとめるよう依頼されたとき
- ユーザが特定の日付の日記について言及したとき（「昨日の日記」「3月1日の日記を書く」など）

## ワークフロー

### 1. 日付の確定
- ユーザが日付を指定している場合はその日付を使う
- 指定がない場合は**前日（日本時間ベース）**の日付を使う。ユーザは翌日に日記を書くのが通常のため
- すべてのツールに日付を渡すときはYYYY-MM-DD形式

### 2. 既存日記の確認
- `diary-drive-tool` で対象日の日記が既に存在するか確認する
- 既に存在する場合は、新規作成ではなく更新を提案する

### 3. 日記テキストの生成
ユーザは最初のメッセージで日記の内容も一緒に渡してくることが多い。その場合はすぐに日記テキストを生成する。内容について深掘りや追加質問はしない。
ユーザが日記を書きたいとだけ言って内容を渡していない場合のみ、内容を促す。

フォーマットのルール：
- ユーザの口調やスタイルを保持する（です/ます、だ/である等は変えない）
- 話題が変わるところで改行を入れて読みやすくする
- 過度な構造化（見出し、箇条書き等）は不要。自然な文章で書く
- 固有名詞（人名、地名、店名等）はユーザの言葉をそのまま使う

### 4. ユーザ確認
- **必ず生成した日記テキストをユーザに見せて確認を取る**
- 修正依頼があれば反映する
- ユーザがOKしたら保存へ進む

### 5. 保存
- 新規の場合：`diary-create-tool` で保存
- 更新の場合：
  1. `diary-drive-tool` で既存の日記を取得
  2. 既存内容に修正・追記を加えた全文を `diary-update-tool` に渡す

### 6. ダイジェスト生成
- 保存が成功したら、`diary-digest-tool` に日付と日記本文を渡してダイジェストを生成・保存する

### 7. 完了報告
- 保存結果をユーザに伝える

## 日記検索時の注意
- `diary-search-tool` で検索する際、1回でヒットしなければクエリや日付範囲を変えて再検索する
- 3回試してもヒットしなければ、それまでにわかったことを返す
