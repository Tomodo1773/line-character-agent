# 日記登録スキル

## スキルの役割

**agentは日記を「書く」「まとめる」役ではない**。ユーザが送ってきた文章を、ほぼそのまま日記としてGoogle Driveに保存することだけがこのスキルの仕事。内容の生成・要約・再構成はしない。ユーザの文章こそが日記本文である。

## いつ使うか

- ユーザが「日記を書きたい」「今日のことを記録して」「昨日の日記に追記して」などと言ったとき
- ユーザが特定の日付の日記について言及しつつ、日記として残したい本文を渡してきたとき（「昨日の日記。〜」など）

## ワークフロー

### 1. 日付の確定
- ユーザが日付を指定している場合はその日付を使う
- 指定がない場合は**前日（日本時間ベース）**の日付を使う。ユーザは翌日に日記を書くのが通常のため
- すべてのツールに日付を渡すときはYYYY-MM-DD形式

### 2. モードの決定
- `diary-drive-tool` で対象日の日記が既に存在するか確認する
- 存在しない → **新規作成モード**（4-A へ）
- 存在する → **追記モード**（4-B へ）
- `diary-search-tool` はこのスキルでは使わない（過去日記をキーワード検索する別用途のツール。作成・追記フローでは不要）

### 3. 日記本文の取り出し

ユーザのメッセージから、ツールに渡す本文を取り出す。**内容の生成・加筆・要約・再構成は一切しない**。

ユーザが最初のメッセージで本文も渡してきている場合はすぐ次のステップに進む。深掘りや追加質問はしない。本文がまったく渡されていない場合のみ、内容を促す。

取り出す際に行ってよい処理は以下の3つのみ。**これ以外の変更は一切しない**。

- **メタ発話の除去**: 日記本文ではない前置きや依頼文を取り除く
  - 例: 「昨日の日記を書きます」「以下の内容で日記作って」「お願いします」
- **不要な句読点の削除**: 連続した句読点や文中で浮いている句読点など、明らかに不要なものだけ取り除く
- **改行の追加**: 話題が変わるところに改行を入れる

**やってはいけないこと**（ユーザが自分でそう書いて送ってきた場合を除く）:

- 見出し（`##` `#` 等）を付けない
- 本文冒頭に日付見出し（`# 2026-04-16` など）を付けない。日付はファイル名で管理されるので本文には不要
- 箇条書き（`-` `*` `1.` 等）にしない
- 内容を要約・言い換え・体裁よく書き直さない
- 「ひとこと」「感想」「まとめ」など、ユーザが書いていないセクションを追加しない
- ユーザの口調やスタイルを変えない（です/ます、だ/である等はそのまま）

### 4. 保存

#### 4-A. 新規作成モード
- セクション3で取り出した本文を `diary-create-tool` に渡す

#### 4-B. 追記モード
- `diary-drive-tool` で取得した既存本文の**末尾に**、セクション3で取り出した本文を連結する
  - 既存本文と新規分の間には空行を1つ挟む
  - 既存本文そのものは編集しない。あくまで末尾への追加のみ（統合・並べ替え・再構成はしない）
- 結合した全文を `diary-update-tool` に渡す

### 5. ダイジェスト生成
- 保存が成功したら、`diary-digest-tool` に日付と保存した日記全文を渡してダイジェストを生成・保存する
- ダイジェストは内部保存用。ユーザへの返信には含めない

### 6. 完了報告
- 保存できたことをユーザに伝える
- ユーザの日記に対して、キャラクターとして一言コメントを添える
